## Fixes applied

- **Server-rendered crawlable link index (`BrowseIndex`).** New `<nav><details>` component (`ui/components/BrowseIndex/`) rendered in the static HTML on four hub pages:
  - Home (`/`, `/fr`): a full A–Z index of all 1,025 Pokémon detail links (the interactive card grid only ships 16 links in the static HTML — the rest were client-only).
  - Type-interactions index (`/type-interactions`, `/fr/type-interactions`): all 171 single/dual-type matchup links (the picker is a client-side filter, so the combo pages had zero internal links).
  - `<details>` keeps it tidy for humans but fully present for crawlers (they read `<details>` content regardless of open state). Plain `<Link>`/`<a>`, no JS gate.
- **`public/_headers`**: `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`, `Referrer-Policy: strict-origin-when-cross-origin`, `Permissions-Policy: camera=(), microphone=(), geolocation=(), browsing-topics=()`. No CSP (Next ships inline hydration + JSON-LD scripts; a strict policy would need `'unsafe-inline'` and buy little — clickjacking is covered by `X-Frame-Options`).
- **`wrangler.jsonc`**: corrected the stale comment that claimed www is the canonical host (the apex is canonical; www 301s to it).

Verification (fresh snapshot build): home = 1,025 pokémon links, type-interactions = 171 combo links, FR mirrors both. BFS over the built output: **orphans 0, click-depth>3 0**, depth histogram `{0:1, 1:1031, 2:1201, 3:171}`. Typecheck clean, 340/340 unit tests pass (incl. new `utils/browseIndex.test.ts`).

## Per-check results

### sitemap-valid
PASS. `sitemap.xml` declared in robots, well-formed, 2,404 `<url>` entries with hreflang alternates. Sampled `<loc>`s (apex `/`) resolve 200; locs use the non-slash canonical form matching `trailingSlash:false`.

PASS. Single self-referencing absolute canonical on every page via `Header` (`constants/Seo.absoluteUrl` + `SITE_ORIGIN`). Verified on home, detail, legal, type-interactions, FR mirrors.

### canonical-resolves-200
PASS. Canonical origin `https://my-pokedex.com/` returns 200 (not a 3xx).

### host-canonicalization
PASS. All four combos reach the canonical apex: `http://apex`→`https://apex` (301, 200); `http://www`→`https://www`→`https://apex` (301 chain); `https://www`→`https://apex` (301). `http://www` takes 2 hops (Cloudflare per-host HTTPS then host redirect) — acceptable, all terminate at apex. Verified by name against a public resolver.

### public-dns-resolution
PASS. `dig @1.1.1.1` — apex and www both return A (104.21.67.218, 172.67.181.102) and AAAA (2606:4700:…). No empty A / black-hole AAAA.

### indexability-directives
PASS. No `noindex` anywhere in the build. The only robots meta is `max-image-preview:large, max-snippet:-1, max-video-preview:-1` (Discover large-image opt-in). No indexability regression since the last commit.

### soft-404-real-status
PASS. `https://my-pokedex.com/<nonexistent>` returns HTTP 404 (Cloudflare `not_found_handling: 404-page`).

### lastmod-honest
🟡 **Open (defensible).** All 2,404 URLs share `lastmod` `2026-07-07`. This is flat, which the check flags — but it is a **manually-pinned constant** (`scripts/generateSitemap.mjs` `LASTMOD`), explicitly *not* `new Date()`, documented to be bumped only when content actually changes, so it never stamps today's date on unchanged URLs across redeploys (the specific dishonest pattern rule 3 warns about). Left unchanged: introducing per-page dates without real per-entity change tracking would be *less* honest, not more. Revisit if/when per-entity modification timestamps exist.
