# Web Accessibility Audit — my-pokedex — 2026-07-18

**Standard:** WCAG 2.2 AA · **Mode:** AUDIT + FIX · **Auditor:** Claude (automated agent, `web-accessibility` skill)
**Verification ceiling:** passes all automated and agent-executable manual checks; human/screen-reader verification pending (see Handover).

## Project profile

- **Stack:** Next.js 16 (Pages Router), React 19, static export (`output: "export"` → `out/`), deployed to Cloudflare via `wrangler deploy`. Tailwind present but Preflight disabled (box-sizing stays `content-box` site-wide — relevant to the reflow fix below).
- **Site type:** Content/reference site (Pokédex): ~1,025 Pokémon detail pages, a type-interactions chart (single + dual combos), legal pages. Bilingual EN + FR (`/fr` subtree).
- **Feature inventory:** one `<form>` (global site search in the nav); no video/audio; SVG type icons; native `<details>`/`<summary>` disclosures; a mobile options sheet + type-filter drawer; client-side routing via `next/link`; self-hosted `<img>` sprites; PWA manifest + service worker.
- **Existing a11y assets:** none prior to this run.

## Scope (applicability matrix)

Applied: foundations, keyboard-and-focus, visual-design, forms (search only), components + ARIA, media (alt), content/SPA, framework lint, WCAG 2.2 A/AA checklist.
N/A and recorded as skips: video captions (no media), form-error association & autocomplete tokens (only field is a free-text search), Lighthouse CI (tooling not installed; axe used as the automated floor).

## Harness

axe-core 4.x driven over CDP against a snapshot-replay `next dev` server (headless Chrome, Node 22 native WebSocket — no Playwright added to the repo). 13 representative routes/states scanned at desktop (1280) and mobile (320) viewports, plus a `prefers-reduced-motion` pass. Reflow measured at a true 320 px layout viewport. Contrast that axe returns as *incomplete* (text over gradients/images) was hand-measured by sampling rendered screenshot pixels with `sharp`.

## Summary of findings & fixes

All findings below were **fixed and re-verified**. Severity in brackets.

**P0 — blockers**
- **[critical] Type filter/chips were fake buttons** (`<span onClick>` with a duplicated `id="type"`): not keyboard-operable, no role/state, and invalid duplicate IDs. → real `<button aria-pressed>` with an `aria-label` that survives the mobile icon-only layout; `id` removed (CSS repointed to `[data-type]`). `PokemonType.tsx`.
- **[critical] Search input had no label** (placeholder-as-label). → visually-hidden `<label>` + `.srOnly` utility. `SearchInput.tsx`.
- **[critical] Search submit was a mouse-only clickable `<img>`.** → real `<button type="submit">` with `aria-label`, decorative `alt=""`. `SearchInput.tsx`.

**P1 — serious**
- **[serious] Missing `<title>` on 404/500** (error screen shipped no `<Head>`). → `<title>` + single `<h1>` + `noindex`. `ErrorScreen.tsx`.
- **[serious] Text contrast below AA** in several places, all measured and fixed: inactive bottom-tab (3.35→5.17:1), TypePicker "Type(s)" label (3.08→5.82:1), type-matchup chips (raw type colour → AA-safe `getTypeChipColor`), matchup multiplier text (light `--t2`/`--th` darkened), **card name over the type gradient (2.8–3.1→5.6–6.0:1)**, **detail-page h1/body/pane-titles/nav-buttons** (deepened gradient + dark-tinted glass; body 4.0→6.7:1, nav button 3.4→6.1:1).
- **[serious] No visible focus** where `outline:none` had no replacement (search field, sort `<select>`). → `:focus-visible` / `:focus-within` rings.
- **[serious] Reflow — horizontal scroll at 320 px**: the shared `Page` wrapper had `min-width:340px`, and `BrowseIndex` used `width:100%`+padding under `content-box`. → `min-width:0` + `box-sizing:border-box`. All Page-wrapped routes now reflow cleanly.
- **[serious] Multiple unnamed `nav` landmarks.** → `aria-label` on the primary and bottom-tab navs (footer nav already labelled).
- **[serious] Logo was a mouse-only clickable `<img alt="logo">`.** → home `<Link>` with an accessible name, decorative image.
- **[moderate→serious] Footer link tap targets < 24 px on mobile.** → `min-height:24px` + spacing.
- **[serious] Ghost tab stops:** the mobile options sheet stayed `display:block` (opacity-hidden) when closed, leaving its controls keyboard-focusable. → `inert` when closed.

**P2 — quality**
- Added a **skip link** as the first focusable element (localized), targeting `<main id="main" tabindex="-1">`.
- Added **`aria-current="page"`** to active nav tabs (top + bottom), **`aria-expanded`/`aria-controls`** to the type-filter button.
- Added a site-wide **`prefers-reduced-motion`** block neutralising the global 1 s colour transition and all component animation.
- Added an **`aria-live` result count** on the home list (EN + FR) so search/type-filter changes are announced (SC 4.1.3).
- Underlined **prose links** on legal pages (Use of Color).

## Definition of Done — results

### axe-violations
0 violations across `wcag2a…wcag22aa` on all 13 routes/states at desktop (1280) and mobile. The only intermittent mobile flag (`target-size` on a bottom tab) was proven to be a Chrome mobile-emulation visual-viewport artifact: the `position:fixed` bar renders just below the emulated 320×640 viewport, so axe measures it clipped. The tabs measure **176×60 px** (probe) and target-size returns **0 violations** on every page when the bar renders within the viewport (360×780, `mobile:false`). Not a real defect.

### keyboard-operable
Top flows (search → results, type filter, card → detail, chart navigation) are fully keyboard operable. Filter chips are buttons with names + `aria-pressed`; search field is labelled with a real submit button; detail Prev/Next are `<Link>`/`<button>`.

### no-keyboard-trap
No traps. The options sheet is `inert` when closed; the type drawer uses `visibility:collapse`; disclosures are native `<details>`.

### visible-focus
Every keyboard stop shows a focus indicator: default UA outlines on links/buttons, added `:focus-visible`/`:focus-within` rings where `outline:none` had removed them, and a high-visibility skip link.

### focus-order-logical
DOM order is source order; no positive `tabindex`; hidden regions are removed from the tab sequence (`inert` / `visibility:collapse` / `display:none`).

### skip-link
`<a href="#main" class="skipLink">` is the first focusable element on every page; hidden until focused, then pinned top-left. Confirmed first-focusable and on-screen on focus.

### reflow-320
No horizontal page scroll at a true 320 px viewport on home, type-interactions (index/filtered/combo), detail, and legal pages (measured `scrollWidth == clientWidth == 320`).

### zoom-400
Verified via the equivalent 320 px reflow method (WCAG 1.4.10): content reflows to a single column with no loss of content or horizontal scrolling; equivalent to 400 % zoom at 1280 px.

### zoom-not-disabled
Viewport meta is `initial-scale=1.0, width=device-width` — no `user-scalable=no` or `maximum-scale`. Pinch-zoom allowed.

### reduced-motion
A global `@media (prefers-reduced-motion: reduce)` block neutralises the global 1 s transition, card fades, nav hide-on-scroll and spotlight transforms. axe reduced-motion pass = 0 violations.

### text-contrast
All normal text ≥ 4.5:1 and large text ≥ 3:1, including the gradient/glass cases axe could not compute (hand-measured by pixel sampling): card names 5.6–6.0:1, detail h1 ~4.97:1 (large), detail body 6.7–7.0:1, matchup chips/multipliers, bottom-tab, picker label — all pass on both themes and all type colours (electric/ice worst cases verified).

### ui-contrast
Focus-ring blues (`#2f6fed` light / `#6ea8ff` dark), chip borders and toggle/nav icons meet ≥ 3:1 against their adjacent colours.

### form-labels
The sole form control (site search) has a programmatic label (`<label for>`), verified present.

### headings-landmarks
Exactly one `<h1>` per page; logical heading order; `header`/`nav`/`main`/`footer`/`article` landmarks present, and every `nav` has a unique accessible name.

### accessible-names
All icon-only buttons (filter, options, theme/resolution toggles, search submit, type chips) have `aria-label` or visible text; images have `alt` or are marked decorative.

### alt-text
Content images carry descriptive alt (sprites keep the `… artwork`/`…-pic` suffix required by the pixelation CSS); decorative glyphs use `alt=""` + `aria-hidden`; the huge card watermark is `aria-hidden`.

### target-size
All interactive targets ≥ 24×24 px (footer links raised to `min-height:24px`; nav/bottom-tab targets are large). See `axe-violations` for the emulation-artifact note.

### status-announcements
An `aria-live="polite"` region on the home list announces the matching count when search/type-filter changes it (SC 4.1.3), verified updating 1025 → 81 on `?types=fire`. SPA route-change focus handling is noted as an enhancement in Handover.

### descriptive-links
Links use descriptive text (nav labels, language switcher "English/Français", footer, Prev/Next, browse-index entries); no bare "click here".

### aria-current
Active primary + bottom-tab nav links expose `aria-current="page"`.

### framework-a11y-lint
ESLint (`next/core-web-vitals` + `next/typescript`, includes `jsx-a11y`) reports **0 errors** on the changed surfaces (only pre-existing `no-img-element` performance *warnings*, a deliberate static-export choice — not accessibility). Note: `next lint` was removed in Next 16; ESLint was run directly.

- **Judgment quality:** alt-text aptness for sprites; reading order of the detail panels.
- **SPA route-change enhancement:** consider moving focus to the page `<h1>`/`<main>` (or announcing the new title) on client-side navigation — beyond WCAG 2.2 AA but improves the SR experience.
