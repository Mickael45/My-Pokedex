# SEO-AUDIT — My Pokédex

Derived SEO guide + standing checklist for this site. Per-run findings and their
dispositions live in the dated run reports (`audits/seo/<date>-report.md`) and the
sidecar (`audits/seo/latest.json`) — this doc is the durable layer and resume point.

**Latest run:** [2026-07-18](audits/seo/2026-07-18-report.md) — orphaned pages 342→0,
click-depth>3 2013→0, security headers added. 1 finding open by design (flat `lastmod`).

## Site profile

- **Stack:** Next.js 16 (Pages Router), static export (`output: "export"`) → `./out`.
- **Host/CDN:** Cloudflare Workers static assets (`wrangler.jsonc`), apex + www bound.
- **Canonical host:** apex `https://my-pokedex.com` (`constants/Seo.SITE_ORIGIN`). www 301s to apex. `trailingSlash:false` — all canonical/hreflang/sitemap URLs are non-slash.
- **Locales:** en (root) + fr (`/fr/*`). Reciprocal en/fr/x-default hreflang; x-default → EN.
- **Content scale:** ~1,025 pokémon detail pages + 171 type-matchup combos, ×2 locales = 2,404 indexable URLs. Data is build-time from a PokéAPI snapshot (record/replay via `POKEDEX_SNAPSHOT`).
- **Ads:** none — AdSense abandoned 2026-07-09 (Nintendo IP), all ad code removed. **Analytics:** none — GA4 + cookie consent removed (commit 7da08ec). No consent UI, no tracking cookies.
- **IP posture:** independent, fan-made, unaffiliated (nominative use); disclaimer on `/about`, `/terms`, footer.

## Where the SEO layer lives

- **Head/meta/OG/JSON-LD/canonical/hreflang:** `ui/components/Header/Header.tsx` (consumes `canonicalPath`, `alternates`, `jsonLd`); origin + defaults in `constants/Seo.ts`.
- **`<html lang>`:** `pages/_document.tsx` `langForPathname`.
- **Sitemap + `_redirects`:** generated by `scripts/generateSitemap.mjs` (pinned `LASTMOD`).
- **Crawlable link hubs:** `ui/components/BrowseIndex/` — server-rendered `<details>` index of all detail + combo links on the home and type-interactions pages (EN/FR). **Do not remove:** this is what keeps orphans at 0 and click-depth ≤3; the visible card grid ships only 16 links and the type picker is client-side.
- **robots / manifest / icons / security headers:** `public/robots.txt`, `public/site.webmanifest`, `public/icons/`, `public/_headers`.

## Standing items (re-check every audit)

- **AI-crawler access** (checked 2026-07-18): robots `Allow: /`, no CDN managed/Content-Signals block; prod robots == repo. All content server-rendered. Retrieval + agent crawlers allowed. Training-crawler blocking is the owner's IP call — not applied.
- **Host canonicalization** (2026-07-18): 4 scheme/host combos all 301 to apex; `http://www` is a 2-hop chain (acceptable). Apex + www resolve A + AAAA on public resolvers.
- **`_headers` honored by Workers assets** — confirm live after each deploy (same mechanism as `_redirects`, which works).
- **Publisher identity / ad-config** — no ads/analytics; if either is ever re-added, re-open the ads + consent scope (ads.txt, publisher ID, CMP, region-scoped Consent Mode).
- **`lastmod` policy** — pinned constant, bumped only on real content change; keep it that way (do not switch to `new Date()`).
- **Fast-movers** — re-verify AI-crawler platform defaults + Cloudflare AI Crawl Control against primary sources on the next audit.

## Currently-active suppressions / open-by-design

- **lastmod-honest** (moderate, open): flat `lastmod` across all URLs is a deliberate, documented pinned constant — see the 2026-07-18 report. Not a defect to "fix" without per-entity change tracking.
