# Image & Navigation Performance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make home-grid images appear faster and make card clicks navigate instantly on normal and slow-3G networks.

**Architecture:** Four cohesive changes — (A) card navigates via prefetched `<Link>`; (B) above-the-fold hero images preloaded with React 19 `ReactDOM.preload`; (C-lite) a free type-color tint placeholder that fades into the image on load; (D) `preconnect`/`dns-prefetch` for third-party ad/analytics origins — plus a build-time PNG→WebP migration. No `next/image` (Vercel Image Optimization is billed).

**Tech Stack:** Next.js 16.2.6 (Pages Router), React 19 (`react-dom` resource APIs), `sharp` (build-time image encoding), CSS Modules.

## Global Constraints

- Pages Router only — no App Router APIs. (Project uses `pages/`.)
- Do NOT add `next/image` or any Vercel Image Optimization usage (cost).
- Do NOT add per-image base64 LQIP / `blurDataURL` to SSG props (payload — home ships all 1025 records).
- WebP only — no PNG fallback, no `<picture>`, no AVIF.
- WebP encoding: `pixel` → `{ lossless: true }`; `basic` and `full` → `{ quality: 82 }`.
- `public/pokemon/` is gitignored and regenerated by `scripts/downloadPokemonImages.mjs` — never commit those assets.
- `<Link>` prefetch only runs in production builds (`next build && next start`), not `next dev` — all prefetch verification must use a production build.
- No test runner exists. Per-task gate = `npx tsc --noEmit` passes + `yarn lint` passes + the stated manual check, then commit.
